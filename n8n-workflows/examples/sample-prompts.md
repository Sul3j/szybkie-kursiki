# Przykładowe prompty do generowania kursów

Kolekcja gotowych promptów, które możesz użyć do generowania kursów za pomocą n8n i Claude.

## Kursy programowania

### Python - Podstawy

```
Stwórz kurs "Python od podstaw" dla początkujących programistów, którzy nie mają wcześniejszego doświadczenia.

Kurs powinien zawierać 5 lekcji:
1. Wprowadzenie do Python - instalacja, pierwszy program
2. Zmienne i typy danych (int, str, float, bool)
3. Instrukcje warunkowe (if, elif, else)
4. Pętle (for, while)
5. Funkcje - definiowanie i wywoływanie

Każda lekcja powinna mieć:
- Szczegółową treść (min 600 słów) z przykładami kodu
- Quiz z 5 pytaniami wielokrotnego wyboru
- Zadanie praktyczne z rozwiązaniem

Używaj prostego języka i praktycznych przykładów.
```

### JavaScript - Modern ES6+

```
Stwórz kurs "JavaScript Modern - ES6+" dla programistów znających podstawy JavaScript.

Tematy:
1. Arrow functions, template literals, destructuring
2. Spread operator, rest parameters
3. Promises i async/await
4. Array methods (map, filter, reduce)
5. Modules (import/export)

Poziom: średniozaawansowany
Każda lekcja z praktycznymi przykładami z prawdziwych projektów.
Quizy i zadania sprawdzające zrozumienie konceptów.
```

### React - Komponenty i Hooki

```
Stwórz kurs "React - Komponenty i Hooki" dla osób znających JavaScript.

Struktura:
1. Wprowadzenie do React - JSX, komponenty funkcyjne
2. Props i State
3. useState i useEffect
4. useContext i useReducer
5. Custom hooks i best practices

Przykłady z budowania prawdziwych komponentów UI.
Zadania praktyczne: budowa todo app, weather app.
```

## Tips dla lepszych promptów

### 1. Bądź konkretny

❌ Zły: "Zrób kurs o Pythonie"
✅ Dobry: "Stwórz kurs Python dla początkujących z 5 lekcjami o podstawach: zmienne, pętle, funkcje, klasy, pliki"

### 2. Określ poziom

Zawsze podaj:
- Dla kogo jest kurs (początkujący/średniozaawansowani/zaawansowani)
- Jaką wiedzę zakładasz u uczestnika

### 3. Wymień tematy lekcji

Jasno określ co ma być w każdej lekcji.

### 4. Określ styl

- Praktyczny vs teoretyczny
- Z przykładami projektów
- Język prosty vs techniczny

### 5. Dodaj kontekst

```
Kurs dla studentów informatyki 2 rok
Kurs dla osób zmieniających karierę na IT
Kurs przygotowujący do certyfikacji
```

## Szablony

### Szablon podstawowy

```
Stwórz kurs "[TYTUŁ]" dla [GRUPA_DOCELOWA].

Kurs powinien zawierać [LICZBA] lekcji:
1. [TEMAT 1]
2. [TEMAT 2]
3. [TEMAT 3]
...

Poziom: [początkujący/średniozaawansowany/zaawansowany]
Styl: [praktyczny/teoretyczny]

Każda lekcja z szczegółową treścią, quizem i zadaniem praktycznym.
```

### Szablon zaawansowany

```
Stwórz [POZIOM] kurs "[TYTUŁ]" dla [GRUPA_DOCELOWA] znających [WYMAGANIA].

Struktura kursu:
1. [TEMAT 1] - [SZCZEGÓŁY]
2. [TEMAT 2] - [SZCZEGÓŁY]
3. [TEMAT 3] - [SZCZEGÓŁY]
4. [TEMAT 4] - [SZCZEGÓŁY]
5. [TEMAT 5] - [SZCZEGÓŁY]

Wymagania wstępne: [LISTA]

Cel kursu: Po ukończeniu kursu uczestnik powinien umieć [CELE].

Każda lekcja powinna zawierać:
- Szczegółową treść (min [SŁOWA] słów)
- Praktyczne przykłady z [KONTEKST]
- Quiz sprawdzający zrozumienie
- Zadanie praktyczne z rozwiązaniem

Styl: [OPIS STYLU]
Przykłady: [RODZAJ PRZYKŁADÓW]
```

## Notatki

- Claude generuje lepsze kursy gdy prompt jest szczegółowy
- Podawaj liczbę lekcji (zalecane: 4-7 dla jednego kursu)
- Określaj długość treści jeśli ważna (min 500-800 słów na lekcję)
- Można prosić o konkretne przykłady (np. "z przykładami e-commerce")
- Warto określić język i ton (formalny, swobodny, dla dzieci, itp.)
